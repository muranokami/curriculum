 ■ ネットワーク層
通信プロトコルの機能や役割を構成する層

* IP(Internet Protocol)
相手先のアドレス情報をもとにしてパケットを送信する機能
インターネット層のプロトコルとなる

* IPヘッダ
IPにおけるデータの送受信単位であるIPパケットの先頭にある、一定の形式に従って制御情報などを記載した部分のこと。

* ICMP
IP通信の疎通状態を調査したり、正常に通信できない場合に送信元ホストエラー情報を返したりする、IP通信を補佐するためのプロトコル

* ICMPメッセージ
問い合わせやエラーに関するメッセージをやり取りすることで、インターネット通信において様々な役割を果たす。

* IPアドレス
インターネット・プロトコルにおいて、通信の相手先を識別するための番号である。

    * オクテット
コンピュータの、特に通信関係の分野などで、厳密には8ビット以外を指すこともある「バイト」の代わりに、必ず8ビットのことを指すものとして使われいる語である。

    * ネットワーク部
そのIPアドレスが属しているネットワークを識別するための部分。

    * ホスト部
そのIPアドレスが属しているホストを識別するための部分。

    * クラス
同様のものの一まとまり。

    * プライベートIPアドレス
家庭内や企業内などの各パソコンに割り振るIPアドレスを指す。

    * グローバルIPアドレス
インターネットに接続する際に利用するIPアドレスです。

* ネットワークアドレス
IPアドレスにおいて、ホスト部のビットが全て０のIPアドレスで、ネットワークそのものを表すアドレスである。

* ブロードキャストアドレス
IPアドレスにおいて、ホスト部のビットが全て1のIPアドレスで、ネットワークそのものを表すアドレスである。

* ループバックアドレス
自分自身を表すアドレス。

* サブネット
IPネットワークを倫理的に細分化したもののことである。

* サブネットマスク
IPアドレスを分割して、どこがネットワークアドレス部分で、どこが端末を表すホストアドレス部分かを識別するために使う数値。

    * プレフィックス表記
先端の制御情報

* デフォルトゲートウェイ
同一ネットワーク内のノードから外部ネットワークに属するノードと通信する際に出入り口となるデバイスやポートのこと
　実務ベースでは、適切な転送先が見つからない場合に使用するIPアドレスを指すことが多い

* IPv6
OSI参照モデルにおいてネットワーク層に位置付けらる通信プロトコルである。

    * サブネットプレフィックス
IPv4におけるネットワーク部に相当する
基本的に前半64ビットがサブネットプレフィックスにあたります

    * インターフェイスID
IPv4におけるホスト部に相当する
 通常のIPv6ノードは、インターフェイスIDをMACアドレスから算出する

* ルータ
コンピュータネットワークにおいて、データを2つ以上の異なるネットワーク間に中継する通信機器
どのルートを通してデータを転送すべきかを判断するルート選択機能を持つ

    * コンソール接続
ルータのコンソールポートとPCをコンソールケーブルで接続する方法

    * VTY接続
物理的に離れたところからネットワークを通じて仮想的に接続する方法

    * AUX接続
ルータのAUXポート（補助ポート）とモデムを接続し、電話回線経由で接続する方法

    * ルーティングテーブル
コンピュータにおけるルーティングテーブルとは、ルーターやネットワーク接続されたコンピュータが持つ、個々のネットワークの宛先への経路の一覧を保持しているテーブル状のデータ構造である。

        * ネクストホップ
次に転送する最も近いルータやインターフェイスのこと

    * ARPテーブル
* IPアドレスとMACアドレスの対応表。

        * ARP
与えられたインターネット層アドレスに対応するリンク層アドレスを発見するために使用される通信プロトコルである。

    * コアルータ
通信キャリアやISPネットワークの通信を中継するために利用されるルータ。

    * センタールータ
通信事業者の回線サービスを使って、本社のネットワークと支店のネットワークを結んだ理、プロバイダと企業ネットを結んでインターネットに繋ぐ。

    * エッジルータ
一つに上流と、一つないし複数の下流ネットワークとだけ接続し、各下流のネットワークへのルートを持たないような場面で使われているルータのこと。

    * ブロードバンドルータ
ルータの中でもインターネット接続を前提として販売されているルータのこと。
 
* ルーティング
異なるネットワークにパケットを送信する際に最適な経路を決めること

    * 直接接続
ルータが直接接続しているネットワークのルータのこと

    * スタティックルーティング
管理者が宛先ネットワークへの最適なルートを手動で設定したルートのこと

    * ダイナミックルーティング
ルーティングプロトコルにより動的に追加されるルートのこと

        * ディスタンスベクタ型
インターフェースの情報を交換することにより、最適経路を決定する方式

        * リンクステート型
インターフェースの情報を交換することにより、最適経路を決定する方式

        * ハイブリッド型
ディスタンスベクタ型をベースに、リンクステート型を組み合わせた方式

 
■ トランスポート層
通信プロトコル（通信手段/通信規約）の機能や役割を階層構造で整理したモデルを構成する層の一つ。

* コネクション型通信
電気通信及びコンピュータネットワークのネットワーク通信モードであり、有用なデータを転送する前に、通信セッションまたは半永久的な接続が確率される。

    * ACK
肯定応答は、テレタイプ端末で使われる制御文字であり、受信側が送信側に対して正しく受信したことを知らせるために送る。

* コネクションレス型通信
経路確保などの事前のやり取り無しに、データ送信を開始するパケット通信方式である。

    * TCP
伝送制御プロトコル

    * スリーウェイハンドシェイク
接続を確立するための手順。

    * Telnet
ネットワークを介して2つのホスト間で通信を行うためのプロトコル。

    * HTTP
データ通信の際に通信内容を暗号化しない方式。

    * FTP
コンピュータネットワーク上のクライアントとサーバ間の間でファイル転送を行うための通信プロトコルの一つである。

    * SMTP
インターネットで電子メールを転送するプロトコル。

* TCPヘッダ
先頭にある制御情報が記された部分のこと。

* UDP
IPネットワーク上のアプリケーション間データグラム送信を実現する通信プロトコルである。

    * RTP
音声や動画などのデータストリームをリアルタイムに配送するためのデータ通信プロトコルである。

    * SNMP
IPネットワーク上のネットワーク機器を監視・制御するための情報の通信方法を定める通信プロトコルである。

    * DHCP
IPアドレス等のネットワークに接続するために必要な情報を自動的に割り当てるプロトコルクライアントサーバ型のプロトコルである

    * TFTP
UDPを用いてコンピュータ間でファイルを転送するためのプロトコルである。

    * RIP
30秒毎に、activeインターフェースからルーティングテーブルについて全ての情報を送信する。

* UDPヘッダ
通信効率を上げるヘッダ。
サイズは８バイト
